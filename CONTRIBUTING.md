# Contributing to FLUXIS

Thanks for your interest in FLUXIS! Here's how you can help.

## Ways to Contribute

**You don't need the source code to contribute meaningfully:**

- 🐛 **Report bugs** — use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) template
- 💡 **Request features** — use the [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md) template
- 📖 **Improve docs** — the README and website docs can always be clearer
- 🧪 **Share programs** — write FLUXIS programs and share them, helps us find edge cases
- ⭐ **Star the repo** — helps more people find FLUXIS
- 📣 **Spread the word** — share on social media, write about it, tell your friends

## Bug Reports

A good bug report includes:
1. The exact `.fx` code that causes the issue
2. The full error output
3. Your OS and FLUXIS version (`fluxis --version`)
4. What you expected vs what happened

## Feature Requests

Before requesting a feature, check if it's already on the [roadmap](README.md#roadmap).
Good feature requests include a concrete use case and example syntax.

## Code of Conduct

Be respectful. We're two 16-year-olds building a programming language on a phone.
Constructive feedback is welcome; hostility is not.
